# gym

Multi-Gym Management SaaS Platform built with Flutter & Firebase

## Overview

This repository hosts a **production-ready** mobile application for a
multi-gym SaaS platform. Four user roles are supported: Super Admin,
Gym Owner, Trainer, and Member. The architecture uses **Clean
Architecture** principles, feature-based folder structure, and the
**MVVM** pattern powered by **Riverpod** for state management.

### Key Technologies

- Flutter with null safety
- Firebase (Authentication, Firestore, Storage, Messaging)
- Riverpod / flutter_riverpod
- Responsive UI with light/dark themes
- Scalable Firestore layout with gymId isolation
- Comprehensive error handling and validation

### Folder Structure Highlights

```
lib/
  core/               # reusable code (models, services, widgets, themes)
  features/           # feature-specific directories
    auth/
    super_admin/
    gym_owner/
    trainer/
    member/
```

Each feature follows the `data/domain/presentation` breakdown described
by Clean Architecture. Providers (view models) live under
`presentation/providers` and screens under `presentation/screens`.

### Firebase Collections

- `users` – holds all user profiles with `role` and optional `gymId`
- `gyms` – records for each subscribed gym
- `members`, `payments`, `expenses`, `workouts`, `dietPlans`, `progress`
  – each document includes `gymId` for multitenancy

### Firestore Rules

Rules are defined in `firestore.rules` demonstrating data isolation and
role-based permissions.

## Running the App

1. Configure Firebase via `firebase_options.dart` (generated by
   `flutterfire` CLI).
2. Run `flutter pub get`.
3. Launch on device/emulator with `flutter run`.
